My Role & Project Objectives

My Role: Core developer responsible for system architecture design, backend microservices development, and blockchain integration.

Project Goal: Create blockchain-based digital passports for each reclaimed wood furniture item, providing immutable supply chain traceability to address industry greenwashing.

Tech Stack: Node.js, Express, PostgreSQL, Polygon Blockchain, IPFS, Docker, AWS

Solution

QR Code Digital Passports

Each furniture item gets a unique QR code linking to blockchain-verified complete supply chain information.

Blockchain Verification

Using Polygon blockchain to ensure data immutability, with IPFS for evidence file storage.

Supply Chain Traceability

Complete GPS-tracked journey from wood reclamation to finished product sales.

Third-party Certification Integration

Connecting with FSC, WWF, and other authoritative organizations to enhance credibility.

âš¡ Technical Challenges & Solutions

Challenge: High Blockchain Transaction Failure Rate

Problem: Initially 30% of on-chain transactions failed due to inaccurate Gas estimation, affecting user experience.

My Solution: Implemented dynamic Gas price estimator + transaction retry queue

Result: Transaction success rate increased to over 95%, user complaints reduced by 80%.

Challenge: IPFS File Access Latency

Problem: Direct file retrieval from IPFS network was slow, impacting passport verification experience.

My Solution: Two-layer caching strategy (Redis memory cache + local file cache)

Result: File loading time reduced from average 2-5 seconds to 300-800 milliseconds.

Challenge: Microservice Data Consistency

Problem: Product state inconsistencies across multiple services led to passport verification failures.

My Solution: Adopted event-driven architecture + Saga pattern for cross-service transactions

Result: Data inconsistency issues reduced by 90%, significantly improving system reliability.
