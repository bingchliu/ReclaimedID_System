3.1 Microservices Architecture

Design Rationale

A microservices architecture was selected to support multi-tenant usage, independent scaling of blockchain and file-processing workloads, and future expansion. Service boundaries were intentionally kept coarse-grained to avoid premature complexity.

Microservices Architecture Diagram

Detailed Service Descriptions

1. Product Service (Product Service)

Node.js + Express + TypeScript

Responsibilities: Product lifecycle management

Database: PostgreSQL (products table)

Core Interfaces:

  • createProduct(): Create new product record
  • generatePassport(): Generate digital passport
  • getProductInfo(): Get product details
  • updateStatus(): Update product status

Key Logic:

  • Validate product data integrity
  • Coordinate with other services to complete passport creation
  • Maintain product state machine

2. Authentication Service (Auth Service)

Node.js + Passport.js + JWT

Responsibilities: User identity authentication and permission management

Cache: Redis (session storage)

Features:

  • Session management
  • API key management

Security Measures:

  • Password hashing: bcrypt
  • Token refresh mechanism
  • Login failure lockout

3. Trace Service (Trace Service)

Python + FastAPI

Responsibilities: Supply chain data tracking and verification

Core Functions:

Node.js + Multer

IPFS Storage Architecture Diagram

File Processing Flow:

  2. Server-side encryption (AES-256-GCM)
  3. Calculate file hash (SHA-256)
  4. Upload to IPFS network
  5. Record IPFS CID to database
  6. Pin file for persistence

  • Automatic image compression
  • Video transcoding optimization

3.3.1 Smart Contract Design

// Digital Passport Smart Contract Core Design
contract FurniturePassport {
    // Key data structures
    struct DigitalPassport {
        string productId;           // Product unique identifier
        bytes32 materialHash;       // Material hash (SHA-256)
        string ipfsEvidenceRoot;    // Evidence IPFS root CID
        address manufacturer;       // Manufacturer address
        address[] certifiers;       // Certification authority list
        uint256 createdAt;          // Creation timestamp
        uint256 lastVerifiedAt;     // Last verification time
        PassportStatus status;      // Passport status
    }

    // Status enumeration
    enum PassportStatus {
        PENDING,    // Pending review
        ACTIVE,     // Active
        VERIFIED,   // Verified
        REVOKED,    // Revoked
        EXPIRED     // Expired
    }

    // Core mappings
    mapping(string => DigitalPassport) public passports;
    mapping(address => bool) public authorizedCertifiers;

    // Key events
    event PassportCreated(
        string indexed productId,
        address indexed manufacturer,
        bytes32 materialHash
    );

    event PassportVerified(
        string indexed productId,
        address indexed certifier,
        uint256 timestamp
    );

    // Core functions
    function createPassport(
        string memory productId,
        bytes32 materialHash,
        string memory ipfsCID
    ) external returns (bool) {
        require(passports[productId].createdAt == 0, "Passport already exists");
        
        passports[productId] = DigitalPassport({
            productId: productId,
            materialHash: materialHash,
            ipfsEvidenceRoot: ipfsCID,
            manufacturer: msg.sender,
            certifiers: new address[](0),
            createdAt: block.timestamp,
            lastVerifiedAt: 0,
            status: PassportStatus.PENDING
        });

        emit PassportCreated(productId, msg.sender, materialHash);
        return true;
    }

    function verifyPassport(string memory productId) external {
        require(authorizedCertifiers[msg.sender], "Not authorized");
        require(passports[productId].createdAt > 0, "Passport not found");
        
        passports[productId].certifiers.push(msg.sender);
        passports[productId].lastVerifiedAt = block.timestamp;
        passports[productId].status = PassportStatus.VERIFIED;

        emit PassportVerified(productId, msg.sender, block.timestamp);
    }
}

3.3.2 Data On-Chaining Strategy

Data On-Chaining Content Diagram

On-chain Data Principles:

  1. Minimization Principle: Only essential data goes on-chain
  2. Hashing Principle: Sensitive data only stores hashes
  3. Cost Optimization: Batch operations to reduce transaction count

Batch Processing Solution:

Using Merkle trees to batch verify multiple product hashes:

  1. Collect a batch of product materialHash values
  2. Build Merkle tree, calculate root hash
  3. Upload root hash to blockchain
  4. Provide Merkle proof for individual verification

Effect: Reduce N transactions to 1 transaction

3.4.1 FSC Certification API Integration

FSC Certification Process:

  1. Brand submits certification application
  2. System calls FSC API to verify qualifications
  3. Obtain temporary verification code
  4. Real-time verification when user scans QR code
  5. Results displayed on passport page

Technical Implementation:

  • Authentication Method: OAuth2.0 client credentials mode
  • Data Synchronization: Webhook real-time notifications + daily batch synchronization
  • Error Handling: Retry mechanism + fallback solution (display "certification pending update")
  • Cache Strategy: Certification results cached for 24 hours

FSC API Integration Example Code

// FSC API Integration Example
class FSCCertificationClient {
    constructor() {
        this.baseURL = 'https://api.fsc.org/v1';
        this.clientId = process.env.FSC_CLIENT_ID;
        this.clientSecret = process.env.FSC_CLIENT_SECRET;
    }

    async verifyCertificate(certificateNumber) {
        // 1. Get access token
        const token = await this.getAccessToken();
        
        // 2. Call verification API
        const response = await axios.get(
            `${this.baseURL}/certificates/${certificateNumber}`,
            {
                headers: {
                    'Authorization': `Bearer ${token}`,
                    'Accept': 'application/json'
                }
            }
        );
        
        // 3. Parse and standardize response
        return this.normalizeResponse(response.data);
    }

    async getAccessToken() {
        // OAuth2.0 client credentials mode
        const response = await axios.post(
            `${this.baseURL}/oauth/token`,
            {
                grant_type: 'client_credentials',
                client_id: this.clientId,
                client_secret: this.clientSecret,
                scope: 'certificate.read'
            }
        );
        
        return response.data.access_token;
    }
}
